## 计划：Python 实现文档向量化完整流程

### 技术选型

1. **嵌入模型调用**：`requests` 库调用 HTTP API（与当前 Java 实现保持一致）
2. **PDF/Docx 解析**：`PyPDF2` 解析 PDF，`python-docx` 解析 Word 文档
3. **向量存储**：使用 `pymysql` 或 `sqlalchemy` 连接 MySQL/SeekDB
4. **JSON 处理**：`json` 库处理向量和元数据的序列化

### 核心模块设计

1. **DocumentParser** - 文档解析器，支持 PDF、Word、TXT 格式
2. **SmartChunker** - 智能语义分块器，实现 512 字符分块、150 重叠、法律条文识别
3. **VectorGenerator** - 批量向量生成器，调用 BAAI/bge-m3 API
4. **VectorStorage** - 向量存储模块，负责数据持久化
5. **PipelineOrchestrator** - 流程编排器，串联各模块

### 输出产物

创建 `scripts/document_vectorization.py` 脚本，包含完整可运行的实现代码。
